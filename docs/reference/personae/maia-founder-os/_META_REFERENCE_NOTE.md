# Why `sandbox/personae/_meta/PERSONA_ARCHITECTURE.md` was not copied here

**LIM-1594, 2026-07-14.** This directory holds the iterable copy of the `maia-founder-os` persona, moved from `liminal-agents/sandbox/personae/maia-founder-os/`. The sibling doc at `liminal-agents/sandbox/personae/_meta/PERSONA_ARCHITECTURE.md` was deliberately **left in place, not copied** here. Reasoning:

1. **It's the multi-persona governing doc, not the Maia persona's own schema.** It documents the shared directory contract (`snapshots/`, `corrections/`, `expected-vault-state.jsonl`, `demo-script.md`) across **five** planned personae, four of which (`priya-iyer-creator`, `lena-park-team`, `darius-okonkwo-business`, `connor-walsh-defense`) don't exist as built directories yet. Copying the whole doc into `liminal-prototype` for one persona would introduce scope and references to personae that aren't here.
2. **It carries an explicit internal-only handling instruction.** Its "Deploy rules — strict" section says: *"Do NOT propagate this insight [the YC-thesis / competitive-positioning cross-reference] as YC-thesis-naming language into any external-facing surface."* `liminal-prototype` is the most externally-visible surface in the workspace (public GitHub Pages demo). Duplicating that document here would be the opposite of what its own deploy rule asks for, independent of whether `liminal-agents` itself is public (a separate, pre-existing condition this move doesn't need to compound).
3. **Its file-path references are `liminal-agents`/`founder-brain`-local.** It cites `lib/agents/*`, `liminal-ip/06-evidence/INVENTIONS.md`, `meta/CONVENTIONS.md`, `meta/CORPUS_MAP.md` — none of which resolve inside `liminal-prototype`. A copy here would be stale on arrival.
4. **The Maia persona is self-sufficient without it.** `README.md` and `identity.json` (both present in this directory) fully specify Maia's character sheet and named-entity registry. `PERSONA_ARCHITECTURE.md` is background on *why* the multi-persona system exists, useful for someone building persona #2, not required to run or understand Maia's own demo.

**Disposition: referenced, not copied.** The canonical location remains `liminal-agents/sandbox/personae/_meta/PERSONA_ARCHITECTURE.md` (prior-art repo, frozen per the LIM-1594 lock note at that path). If a future persona ships in `liminal-prototype` and the multi-persona contract becomes load-bearing here too, that's the trigger to reconsider — at that point copy or re-derive a `liminal-prototype`-scoped version rather than porting the founder-brain-referencing original verbatim.
